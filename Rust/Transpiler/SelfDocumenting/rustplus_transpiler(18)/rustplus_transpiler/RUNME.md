# RUNME — Normal Cargo Workflow

The intended Rust Plus run experience is:

```text
write .rp files
run cargo build / cargo run
build.rs generates sibling .rs files
rustc compiles the generated .rs files normally
```

You should not manually run `rustplus transpile` during normal development.

## 1. Requirements

Install Rust if needed:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Verify:

```bash
rustc --version
cargo --version
rustfmt --version
```

## 3. The key project files

### `Cargo.toml`

```toml
[package]
name = "normal-rust-project"
version = "0.1.0"
edition = "2021"
build = "build.rs"

[build-dependencies]
rustplus = { path = "../.." }

[[bin]]
name = "normal-rust-project"
path = "src/main.rs"
```

The explicit `[[bin]]` target is important when your entry file starts as `src/main.rp`. It tells Cargo the generated binary target path will be `src/main.rs`.

### `build.rs`

```rust
fn main() {
    rustplus::cargo_integration::compile_sibling_files("src")
        .expect("failed to generate Rust files from .rp files");
}
```

### `src/main.rp`

```rust
mod account;

use account::{Account, IAccount};

fn main()
{
    let mut account: IAccount = Account::Heap("account-1");

    account.deposit(250);
    account.deposit(125);

    println!("{}", account.id());
    println!("balance = {}", account.balance());
}
```

Generated `src/main.rs`:

```rust
// @generated by rustplus. Do not edit manually.
// Source: src/main.rp

mod account;

use account::{Account, IAccount};

fn main() {
    let mut account: Box<dyn IAccount> = Box::new(Account::new("account-1".to_string()));

    account.deposit(250);
    account.deposit(125);

    println!("{}", account.id());
    println!("balance = {}", account.balance());
}
```

### `src/account.rp`

```rust
pub interface IAccount
{
    fn id(&self) -> &str;
    fn deposit(&mut self, amount: i64);
    fn balance(&self) -> i64;
}

pub class Account : IAccount
{
    id: String;
    balance: i64;

    pub fn new(id: String) -> Self
    {
        return Self
        {
            id,
            balance: 0,
        };
    }

    pub fn id(&self) -> &str
    {
        return &self.id;
    }

    pub fn deposit(&mut self, amount: i64)
    {
        self.balance += amount;
    }

    pub fn balance(&self) -> i64
    {
        return self.balance;
    }
}
```

## 4. Add `.rp` files to your own Cargo project

Add this to your `Cargo.toml`:

```toml
[package]
build = "build.rs"

[build-dependencies]
rustplus = { path = "../rustplus_transpiler" }
```

If you want `src/main.rp` instead of a hand-written `src/main.rs`, add an explicit binary target:

```toml
[[bin]]
name = "my-app"
path = "src/main.rs"
```

Add `build.rs`:

```rust
fn main() {
    rustplus::cargo_integration::compile_sibling_files("src")
        .expect("failed to generate Rust files from .rp files");
}
```

Create Rust Plus files:

```text
src/main.rp
src/payment.rp
```

Use modules normally:

```rust
mod payment;
```

Run normal Cargo commands:

```bash
cargo check
cargo run
cargo test
cargo build --release
```

## 5. Interface object construction

Rust Plus does not use C#-style declarations as the standard surface syntax. Use Rust-style `let` declarations with explicit `Heap` construction when assigning a concrete class to an interface variable.

Supported `.rp` syntax for concrete stack-owned values:

```rust
let account: Account = Account::Stack("account-1");
```

Generated Rust:

```rust
let account: Account = Account::new("account-1".to_string());
```

Supported `.rp` syntax for interface heap-owned values:

```rust
let account: IAccount = Account::Heap("account-1");
```

Generated Rust:

```rust
let account: Box<dyn IAccount> = Box::new(Account::new("account-1".to_string()));
```

If the interface has mutable methods, make the binding mutable:

```rust
let mut account: IAccount = Account::Heap("account-1");
```

Generated Rust:

```rust
let mut account: Box<dyn IAccount> = Box::new(Account::new("account-1".to_string()));
```

Rejected by default:

```rust
IAccount account = new Account("account-1");
```

## 6. Stack / Heap initializers

Rust Plus supports explicit stack-owned and boxed heap-owned initialization:

```rust
let article: Article = Article::Stack("My own new!", "Lol!", "stuff");
let boxed_article: Box<Article> = Article::Heap("My own new!", "Lol!", "stuff");
let account: Account = Account::Stack("account-1");
let interface_account: IAccount = Account::Heap("account-1");
```

Generated Rust:

```rust
let article: Article = Article::new("My own new!", "Lol!", "stuff");
let boxed_article: Box<Article> = Box::new(Article::new("My own new!", "Lol!", "stuff"));
let account: Account = Account::new("account-1".to_string());
let interface_account: Box<dyn IAccount> = Box::new(Account::new("account-1".to_string()));
```

The constructor must use standard Rust Plus/Rust method syntax:

```rust
pub fn new(headline: &str, location: &str, artist: &str) -> Self
```

`pub new(...)` is intentionally not supported.

Inspect the included example:

```bash
cd ../..
cargo run -- check examples/stack_heap_article.rp
cargo run -- check examples/account_stack_heap.rp
```

## 7. Generated file safety

Rust Plus writes generated sibling files with this marker:

```rust
// @generated by rustplus. Do not edit manually.
```

By default it refuses to overwrite a hand-written `.rs` file that does not contain that marker.

Safe:

```text
src/account.rp  -> src/account.rs generated by rustplus
src/main.rp     -> src/main.rs generated by rustplus
```

Unsafe and rejected by default:

```text
src/account.rp
src/account.rs  # manually written non-generated file
```

## 8. Disable rustfmt in build.rs

Use options if you want faster generation:

```rust
fn main() {
    let options = rustplus::cargo_integration::CargoBuildOptions::new("src")
        .without_rustfmt();

    rustplus::cargo_integration::compile_sibling_files_with_options(options)
        .expect("failed to generate Rust files from .rp files");
}
```

## 11. VS Code syntax highlighting for `.rp`

The repository includes workspace settings that force `.rp` files to use the `rustplus` language ID:

```json
{
    "files.associations": {
        "*.rp": "rustplus"
    }
}
```

These settings are included in:

```text
.vscode/settings.json
examples/normal-rust-project/.vscode/settings.json
rustplus.code-workspace
```

Important: the setting only selects the language ID. The local grammar still has to be installed as a VS Code extension.

### Option A — use the included VS Code task

Open the repository root in VS Code:

```bash
code .
```

Run the task:

```text
Terminal -> Run Task -> Rust Plus: Install VS Code Extension
```

Then reload VS Code:

```text
Developer: Reload Window
```

Open any `.rp` file. The lower-right language mode should show:

```text
Rust Plus
```

### Option B — install manually

From the repository root:

```bash
cd tooling/vscode-rustplus
npx --yes @vscode/vsce package --allow-missing-repository

```

Reload VS Code and open:

```text

```

This is useful while editing the grammar itself:

```bash

```

Press `F5`, then open a `.rp` file in the Extension Development Host.

### Troubleshooting

### Build error: `Cannot bind Account::Stack(...) to interface type IAccount`

This means a `.rp` file tried to bind a concrete stack-owned value to an interface variable:

```rust
let account: IAccount = Account::Stack("account-1");
```

Use one of these two valid forms instead:

```rust
let account: Account = Account::Stack("account-1");
let account: IAccount = Account::Heap("account-1");
```

`Stack` means direct owned value. `Heap` means boxed value. Interface variables lower to `Box<dyn Interface>`, so they must use `Heap`.


If highlighting still does not work:

1. Open a `.rp` file.
2. Click the language mode in the lower-right corner.
3. Select `Rust Plus`.
4. Run `Developer: Reload Window`.
5. Confirm the extension is installed under Extensions with the name `Rust Plus Syntax`.

If the language mode only shows `Plain Text`, the extension is not installed or VS Code has not been reloaded after installation.

## TypeScript-inspired compiler architecture

Rust Plus now includes a project-level compiler object, `RustPlusProgram`, inspired by the useful parts of TypeScript's `Program` / `SourceFile` architecture.

Added pieces:

```text
src/program.rs        # project compiler object
src/host.rs           # file-system abstraction + memory host
src/diagnostics.rs    # structured diagnostics
src/line_map.rs       # approximate .rp -> .rs line maps
src/incremental.rs    # Cargo build cache
src/timing.rs         # parse/bind/validate/emit timings
ARCHITECTURE.md       # detailed architecture notes
```

Useful commands:

```bash
cargo run -- check examples/account.rp --diagnostics
cargo run -- check examples/account.rp --extended-diagnostics
```

Cargo-native builds now use an incremental cache under:

```text
target/rustplus/cache.toml
```

Generated sibling files also get `.rpmap` sidecar files, for example:

```text
src/main.rs.rpmap
src/account.rs.rpmap
```

See `ARCHITECTURE.md` for the full model.
